## 🤝 コントリビューション方法

### バグ報告

バグを見つけた場合は、以下の情報を含めてIssueを作成してください：

- **環境情報**: OS、Pythonバージョン、Node.jsバージョン
- **再現手順**: バグを再現するための具体的な手順
- **期待される動作**: 本来どう動作すべきか
- **実際の動作**: 実際に何が起こったか
- **スクリーンショット**: 可能であれば添付

### 機能提案

新機能のアイデアがある場合：

1. まずIssueで提案内容を説明してください
2. コミュニティの意見を聞きましょう
3. 承認されたら実装を開始してください

### Pull Request

1. **Fork** このリポジトリをForkします
2. **ブランチ作成** 機能ブランチを作成します
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **変更を加える** コードを修正します
4. **テスト** 変更が既存機能を壊していないか確認
5. **コミット** わかりやすいコミットメッセージで
   ```bash
   git commit -m "Add: 新機能の説明"
   ```
6. **Push** フォークしたリポジトリにPush
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Pull Request** 本リポジトリに対してPRを作成

## 📝 コーディング規約

### Python

- PEP 8スタイルガイドに従う
- 関数とクラスにdocstringを記述
- 型ヒントを使用（Python 3.8+）

```python
def fetch_data(url: str, timeout: int = 10) -> dict:
    """
    指定されたURLからデータを取得する
    
    Args:
        url: 取得先のURL
        timeout: タイムアウト時間（秒）
    
    Returns:
        取得したデータの辞書
    """
    pass
```

### JavaScript/React

- ESLintの推奨ルールに従う
- 関数コンポーネントとHooksを使用
- PropTypesまたはTypeScriptで型を定義

```javascript
function ComponentName({ prop1, prop2 }) {
  // コンポーネントロジック
  return (
    <div>
      {/* JSX */}
    </div>
  );
}
```

## 🧪 テスト

変更を加えた場合は、以下を確認してください：

- [ ] スクレイパーが正常に動作する
- [ ] APIサーバーが起動する
- [ ] フロントエンドが表示される
- [ ] 既存機能が壊れていない
